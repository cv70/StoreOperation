# ROI 测算模板

## 1. 文档目标

ROI 文档的目的不是做财务展示，而是统一试点、转正和扩店时的价值口径，避免“试点有效但无法签约”的常见问题。

本模板统一回答三个问题：

1. 我们到底创造了多少业务价值
2. 这部分价值中哪些可以合理归因给本产品
3. 这些价值是否足以支撑年度预算和扩店

---

## 2. ROI 总公式

`ROI = (增量毛利 + 降本收益 + 风险损失减少 - 项目总成本) / 项目总成本`

试点期建议按周追踪，8 周验收，年度转正时再做 12 个月外推。

---

## 3. 收益结构

## 3.1 增量毛利

主要来自：

- 缺货降低带来的销售回补
- 活动优化带来的有效增量销售
- 菜单或陈列优化带来的结构改善

建议计算：

`增量毛利 = (回补销售额 + 活动增量销售额) x 毛利率`

## 3.2 降本收益

主要来自：

- 报损下降
- 人效优化
- 可选的能耗优化

建议计算：

`降本收益 = 报损减少金额 + 人效收益 + 可选其他降本项`

## 3.3 风险损失减少

主要来自：

- 舞弊和异常退款损失下降
- 高危事件处理更快带来的损失规避
- 设备或安全事件的损失减少

---

## 4. 成本结构

项目总成本建议统一包含：

- 试点服务费或年度订阅费
- 实施费用
- 培训费用
- 客户内部投入成本

如果在试点阶段刻意漏算客户内部投入，后续转正时 ROI 很容易被挑战。

---

## 5. 归因原则

ROI 最大争议通常不在公式，而在归因。

建议固定以下原则：

## 5.1 先冻结基线

- 试点前建议选取 4 周基线窗口
- 试点门店和对照门店在立项时冻结

## 5.2 能对照就尽量对照

优先级建议：

1. 对照门店
2. 对照时段
3. 历史基线

## 5.3 分场景记账

不要把所有改善混成一个总数，必须分场景追踪：

- 补货场景收益
- 临期场景收益
- 告警控险收益
- 排班优化收益
- 活动 ROI 收益

这样试点结束后，客户才能清楚知道下一年该买哪些包。

---

## 6. 核心指标口径

## 6.1 缺货率

`缺货率 = 缺货SKU数 / 在售SKU总数`

## 6.2 报损率

`报损率 = 报损金额 / 领用金额`

## 6.3 严重告警响应时长

`严重告警响应时长 = 首次触达到首次处理耗时`

## 6.4 建议采纳率

`建议采纳率 = 已采纳建议数 / 总建议数`

## 6.5 任务按时完成率

`任务按时完成率 = 按时完成任务数 / 全部到期任务数`

## 7. 输入字段模板

| 字段 | 说明 |
|---|---|
| brand_id | 品牌 ID |
| region_id | 区域 ID |
| store_id | 门店 ID |
| business_week | 业务周 |
| stockout_recovered_sales | 缺货回补销售额 |
| promo_incremental_sales | 活动增量销售额 |
| gross_margin_rate | 毛利率 |
| loss_reduction_amount | 报损减少金额 |
| labor_efficiency_gain | 人效收益 |
| risk_loss_reduction | 风险损失减少 |
| subscription_fee | 订阅费用 |
| implementation_fee | 实施费用 |
| training_fee | 培训费用 |
| internal_hr_cost | 客户内部投入成本 |

---

## 8. 计算步骤

1. 冻结试点门店、范围和口径
2. 采集并确认基线数据
3. 按场景、按周计算收益
4. 汇总项目成本
5. 形成 8 周 ROI 结论
6. 做 12 个月审慎、中性、积极三档外推

---

## 9. 年度外推建议

| 档位 | 覆盖率 | 效果折算 |
|---|---|---|
| 审慎 | 40% 门店覆盖 | 按试点效果的 60% |
| 中性 | 70% 门店覆盖 | 按试点效果的 80% |
| 积极 | 100% 门店覆盖 | 按试点效果的 100% |

转正谈判时建议默认采用“中性”方案，避免销售承诺过度乐观。

---

## 10. 结果输出模板

试点结束时建议固定输出以下内容：

1. 8 周总 ROI
2. 每个场景的单独收益贡献
3. Top 门店改善案例
4. 低表现门店原因
5. 年度扩店 ROI 预测

---

## 11. 常见争议与解决办法

| 争议 | 原因 | 解决办法 |
|---|---|---|
| 改善来自季节性，不来自产品 | 缺少对照 | 使用对照门店或对照时段 |
| 活动、天气、节假日影响混杂 | 多因素叠加 | 按场景分账，不做混合归因 |
| 客户说内部投入被低估 | 成本没算全 | 把客户内部人天成本纳入 |
| 口径中途变更 | 立项不严谨 | 立项冻结并双方法务确认 |

---

## 12. 结论

ROI 模型的目标不是让数字看起来更大，而是让客户更容易批准预算、更容易签年度合同、更容易支持扩店。
